# Notes App

A simple notes app: create, view, edit, and delete notes.

- **Frontend:** React (Vite) + Tailwind CSS
- **Backend:** FastAPI (Python), async via Motor
- **Database:** MongoDB Atlas

```
notes-app/
├── backend/
│   ├── app/
│   │   ├── main.py           # FastAPI app + CORS + startup/shutdown
│   │   ├── config.py         # Settings loaded from .env
│   │   ├── database.py       # MongoDB Atlas connection (Motor)
│   │   ├── schemas.py        # Pydantic request/response models
│   │   └── routes/
│   │       └── notes.py      # CRUD endpoints: GET/POST/PUT/DELETE /api/notes
```

## 0. Prerequisites

Install these once if you don't already have them:

- **Node.js** 18+ — check with `node -v`
- **Python** 3.11+ — check with `python3 --version`
- **VS Code** with the **Python** and **ES7+ React/Redux/JS snippets** extensions (optional but helpful)
- A free **MongoDB Atlas** account (you said you already have one — see step 1)

---

## 1. Get your MongoDB Atlas connection string

1. Log in to [MongoDB Atlas](https://cloud.mongodb.com).
2. Pick your cluster → **Connect** → **Drivers** → select **Python**.
3. Copy the connection string. It looks like:
   `mongodb+srv://<username>:<password>@<cluster>.mongodb.net/?retryWrites=true&w=majority`
4. In Atlas, go to **Database Access** and make sure you have a database user with a username/password (create one if needed). If the password has special characters (`@`, `#`, `%`, etc.), URL-encode them.
5. Go to **Network Access** and add your current IP (or `0.0.0.0/0` for "allow from anywhere" while developing — tighten this later).
6. You don't need to manually create the `notes_app` database or `notes` collection — MongoDB creates both automatically the first time a note is saved.

---

## 2. Open the project in VS Code

```bash
cd notes-app
code .
```

Open **two terminals** in VS Code (Terminal → Split Terminal) — one for the backend, one for the frontend.

---

## 3. Backend setup (Terminal 1)

```bash
cd backend

# Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Create your real .env file from the example
cp .env.example .env            # Windows: copy .env.example .env
```

Open `backend/.env` and paste in your real Atlas connection string:

```
MONGODB_URI=mongodb+srv://yourUser:yourPassword@yourCluster.mongodb.net/?retryWrites=true&w=majority
DATABASE_NAME=notes_app
CORS_ORIGINS=http://localhost:5173
```

Run the API:

```bash
uvicorn app.main:app --reload --port 8000
```

Visit **http://localhost:8000/api/health** — you should see `{"status":"ok"}`.
Interactive API docs (auto-generated by FastAPI) are at **http://localhost:8000/docs**.

If the server fails to start with a Mongo connection/auth error, double-check your username, password, and that your IP is allow-listed in Atlas Network Access.

---

## 4. Frontend setup (Terminal 2)

```bash
cd frontend

# Install dependencies
npm install

# Create your real .env file
cp .env.example .env            # Windows: copy .env.example .env
```

`frontend/.env` should point at your local backend (already set correctly by default):

```
VITE_API_URL=http://localhost:8000
```

Run the dev server:

```bash
npm run dev
```

Visit **http://localhost:5173** — the Notes app should load, and creating/editing/deleting a note should immediately hit your MongoDB Atlas cluster.

---

## 5. Quick verification

1. Click **+ New note**, add a title and some text, save.
2. Refresh the page — the note should still be there (confirms it saved to Atlas, not just local state).
3. In Atlas, go to **Browse Collections** on your cluster — you should see a `notes_app` database with a `notes` collection containing your document.
4. Try editing and deleting a note from the UI to confirm the full CRUD loop works.

## API reference

| Method | Endpoint             | Description       |
|--------|-----------------------|--------------------|
| GET    | `/api/notes`          | List all notes     |
| POST   | `/api/notes`          | Create a note      |
| GET    | `/api/notes/{id}`     | Get one note       |
| PUT    | `/api/notes/{id}`     | Update a note      |
| DELETE | `/api/notes/{id}`     | Delete a note      |

Request/response body for create/update:
```json
{ "title": "Groceries", "content": "Milk, eggs, coffee" }
```

## Notes on going further

- **Auth:** there's no login yet — anyone with the URL can see/edit all notes. Fine for personal/local use; add auth (e.g. JWT + a users collection) before deploying publicly.
- **Deploying:** the backend can go on Render/Railway/Fly.io, the frontend on Vercel/Netlify. Just update `CORS_ORIGINS` in the backend `.env` and `VITE_API_URL` in the frontend `.env` to the deployed URLs.
- **Indexes:** if the notes list grows large, add a MongoDB index on `updated_at` for faster sorting (`db.notes.createIndex({ updated_at: -1 })` in the Atlas shell).
